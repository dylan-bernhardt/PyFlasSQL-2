Bienvenue dans notre projet

Ce projet est conçu pour illustrer et expliquer des failles de sécurité courantes dans les applications web. À travers des exemples interactifs et des démonstrations, les utilisateurs peuvent apprendre comment ces failles sont exploitées et comment les prévenir.

Les Failles de Sécurité

Dans ce projet, nous explorons trois failles de sécurité majeures qui ont été volontairement implémentées, chacune présentée avec des détails techniques et des exemples pratiques.

  1. Injection SQL

    Cette faille permet à un attaquant d'injecter des commandes SQL malveillantes à travers l'interface utilisateur, affectant ainsi la base de données sous-jacente. Nous démontrons comment ces attaques sont menées et comment les prévenir.

  2. Failles Cryptographiques

    Cette section explore les faiblesses dans l'implémentation de la cryptographie, telles que l'utilisation de hachages MD5 obsolètes. Nous montrons comment ces faiblesses peuvent être exploitées et comment utiliser des méthodes de cryptographie plus sûres.

  3. Command execution

    Cette faille permet d'injection du code bash directement sur le système d'exploitation du serveur. Nous modélisons cette faille via un formulaire permettant à l'utilisateur de scanner un ordinateur sur son réseau à l'aide de la commande nmap.
